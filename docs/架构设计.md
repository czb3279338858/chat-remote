# 移动端聊天控制多类型实例系统 —— 架构设计

## 1. 架构概述

### 1.1 设计目标
- 支持通过手机聊天软件（飞书）统一控制 PC 上的多种实例
- 实现指令简洁、异常提示明确、全程无鉴权、纯透传交互
- 支持交互式和非交互式两种运行模式
- 预留类型扩展能力，便于后续添加新实例类型

### 1.2 架构风格
采用**分层架构 + 插件化设计**：
- 分层架构：职责分离，便于维护和测试
- 插件化设计：实例类型可动态扩展

## 3. 核心模块设计

### 3.1 接入层 (Adapter Layer)

**职责**：对接不同聊天平台的消息收发

**核心接口**：
```typescript
interface IMessageAdapter {
  // 发送消息到聊天平台
  sendMessage(message: OutgoingMessage): Promise<void>;
  // 注册消息接收回调
  onMessage(callback: (message: IncomingMessage) => void): void;
  // 初始化连接
  initialize(): Promise<void>;
}
```

**默认实现**：飞书 Bot 适配器

### 3.2 指令层 (Command Layer)

**3.2.1 指令解析器 (CommandParser)**
- 解析用户输入，提取指令名和参数
- 支持容错处理：多空格合并、大小写转小写

**3.2.2 指令分发器 (CommandDispatcher)**
- 根据指令名路由到对应的处理器

**3.2.3 指令处理器接口**：
```typescript
/**
 * 指令处理器接口
 * 所有指令处理器必须实现此接口，用于处理用户输入的指令
 */
interface ICommandHandler {
  /** 指令名称，如 'help', 'new', 'list' 等 */
  readonly name: string;
  
  /** 指令功能描述，用于帮助信息展示 */
  readonly description: string;
  
  /** 指令使用示例，如 '/new <类型> <备注> [参数]' */
  readonly usage: string;
  
  /**
   * 执行指令
   * @param args - 指令参数列表，已去除指令名本身
   * @param context - 指令执行上下文，包含当前会话信息、默认实例等
   * @returns Promise<CommandResult> - 指令执行结果
   */
  execute(args: string[], context: CommandContext): Promise<CommandResult>;
}

/**
 * 指令执行上下文
 * 包含执行指令时所需的上下文信息
 */
interface CommandContext {
  /** 当前会话/用户标识 */
  sessionId: string;
  /** 当前设置的默认实例ID，未设置时为 undefined */
  defaultInstanceId?: number;
  /** 消息发送者信息 */
  sender: SenderInfo;
  /** 原始消息对象 */
  rawMessage: IncomingMessage;
}

/**
 * 指令执行结果
 */
interface CommandResult {
  /** 是否执行成功 */
  success: boolean;
  /** 返回给用户的消息内容 */
  message: string;
  /** 可选的附加数据 */
  data?: any;
}
```

**支持的指令**：
| 指令 | 处理器类 | 说明 |
|------|----------|------|
| /help | HelpCommandHandler | 帮助信息 |
| /new | NewCommandHandler | 创建实例 |
| /list | ListCommandHandler | 列出实例 |
| /log | LogCommandHandler | 查看历史 |
| /rename | RenameCommandHandler | 修改备注 |
| /use | UseCommandHandler | 设置默认实例 |
| /kill | KillCommandHandler | 关闭实例 |
| /killall | KillAllCommandHandler | 批量关闭 |
| /interrupt | InterruptCommandHandler | 中断实例 |
| /<ID> | SendCommandHandler | 发送消息 |
| 纯文本 | DefaultSendHandler | 发送给默认实例 |

### 3.3 核心层 (Core Layer)

**3.3.1 实例管理器 (InstanceManager)**
- 管理所有运行中实例的生命周期
- 分配自增 ID
- 提供实例 CRUD 操作

```typescript
/**
 * 实例管理器接口
 * 负责管理所有运行中实例的生命周期，提供实例的增删改查(CRUD)操作
 */
interface IInstanceManager {
  /**
   * 创建新实例
   * @param type - 实例类型，如 'qodercli'
   * @param remark - 用户自定义备注，用于识别实例用途
   * @param params - 启动参数数组，如 ['-w=/path/to/project']
   * @returns Promise<Instance> - 返回创建的实例对象
   * @throws 当类型不支持或参数验证失败时抛出异常
   */
  create(type: string, remark: string, params: string[]): Promise<Instance>;
  
  /**
   * 根据ID获取实例
   * @param id - 实例唯一标识（自增数字）
   * @returns Instance | undefined - 返回实例对象，不存在时返回 undefined
   */
  get(id: number): Instance | undefined;
  
  /**
   * 获取所有运行中的实例
   * @returns Instance[] - 返回所有状态不为 EXITED 的实例数组
   */
  getAll(): Instance[];
  
  /**
   * 关闭指定实例
   * @param id - 要关闭的实例ID
   * @returns Promise<void>
   * @throws 当实例不存在或已退出时抛出异常
   */
  kill(id: number): Promise<void>;
  
  /**
   * 批量关闭实例
   * @param type - 可选，指定要关闭的实例类型；不传则关闭所有实例
   * @returns Promise<void>
   */
  killAll(type?: string): Promise<void>;
}
```

**3.3.2 状态管理器 (StatusManager)**
- 管理实例状态流转
- 状态：IDLE(空闲) / BUSY(忙碌) / EXITED(已退出)

**3.3.3 进程管理器 (ProcessManager)**
- 封装子进程操作
- 管理 stdin/stdout/stderr
- 处理进程启动、通信、终止

**3.3.4 消息分块处理器 (ChunkHandler)**
- 按预设长度阈值分割长消息
- 添加顺序标记 [1]、[2]、[完成]

**3.3.5 历史记录器 (HistoryLogger)**
- 记录已退出实例信息
- 默认保留最近 100 条

### 3.4 实例类型层 (Type Layer)

**实例类型定义接口**：
```typescript
interface IInstanceType {
  readonly name: string;
  readonly description: string;
  readonly runMode: RunMode;           // INTERACTIVE / NON_INTERACTIVE
  readonly interruptible: boolean;      // 是否可中断
  readonly defaultParams: string[];     // 默认启动参数
  
  // 验证启动参数
  validateParams(params: string[]): ValidationResult;
  // 创建进程启动配置
  createProcessConfig(params: string[]): ProcessConfig;
  // 处理实例输出
  handleOutput(data: Buffer, instance: Instance): string;
}
```

**类型注册中心 (TypeRegistry)**：
- 管理所有支持的实例类型
- 提供类型查询和验证功能

### 3.5 数据模型

**Instance (实例)**：
```typescript
interface Instance {
  id: number;                    // 自增数字 ID
  type: string;                  // 实例类型
  remark: string;                // 用户备注
  params: string[];              // 启动参数
  runMode: RunMode;              // 运行模式
  status: InstanceStatus;        // 运行状态
  features: Feature;             // 实例特性
  createdAt: Date;               // 启动时间
  process: ChildProcess;         // 进程引用
  outputBuffer: string;          // 输出缓冲（用于分块）
}
```

**RunMode (运行模式)**：
```typescript
enum RunMode {
  INTERACTIVE = 'interactive',      // 交互式
  NON_INTERACTIVE = 'non_interactive' // 非交互式
}
```

**InstanceStatus (实例状态)**：
```typescript
enum InstanceStatus {
  IDLE = 'idle',        // 运行中-空闲
  BUSY = 'busy',        // 运行中-忙碌
  EXITED = 'exited'     // 已退出
}
```

**Feature (实例特性)**：
```typescript
enum Feature {
  INTERRUPTIBLE = 'interruptible',       // 可中断
  NON_INTERRUPTIBLE = 'non_interruptible' // 不可中断
}
```

**HistoryRecord (历史记录)**：
```typescript
interface HistoryRecord {
  originalId: number;        // 原实例 ID
  type: string;              // 实例类型
  remark: string;            // 用户备注
  exitTime: Date;            // 退出时间
  exitReason: ExitReason;    // 退出原因
}
```

## 4. 数据流设计

### 4.1 创建实例流程 (/new)

```
用户输入 → 飞书 Bot → 指令解析 → 类型验证 → 创建实例
                                            ↓
返回创建结果 ← 发送初始输出 ← 捕获初始输出 ← 启动进程
                                            ↓
                                      状态: 忙碌 → 空闲
```

### 4.2 发送消息流程 (/<ID> 或纯文本)

```
用户输入 → 指令解析 → 查找实例 → 验证状态 → 写入 stdin
                                            ↓
返回分块响应 ← 分块处理 ← 累积输出 ← 监听 stdout
```

### 4.3 实例退出流程

```
进程退出事件 → 更新状态为 EXITED → 移出活跃列表
                                            ↓
                                   记录到历史日志
```

## 5. 关键设计决策

### 5.1 无鉴权设计
- 系统内部不做任何身份验证
- 依赖聊天平台自身的权限控制
- 简化使用流程，符合需求

### 5.2 纯透传设计
- 消息内容不做业务处理
- 直接转发给目标实例的 stdin
- 实例输出直接返回给用户

### 5.3 响应分块机制
- 设置消息长度阈值（如 2000 字符）
- 超过阈值时自动分块发送
- 每块添加顺序标记

### 5.4 扩展性设计
- 实例类型通过接口定义，可动态注册
- 指令处理器通过注册表管理，可动态添加
- 接入层通过适配器模式，支持多平台

## 8. 异常处理策略

| 层级 | 处理策略 |
|------|----------|
| 接入层 | 网络异常重连，消息发送失败提示 |
| 指令层 | 解析失败返回格式错误提示 |
| 核心层 | 进程异常捕获，自动清理资源 |
| 类型层 | 参数验证失败返回具体错误 |

所有异常提示遵循统一模板：**错误原因 + 解决方案**
