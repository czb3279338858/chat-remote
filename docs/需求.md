# 移动端聊天控制多类型实例系统 —— 需求规格说明书

## 1. 项目概述

### 1.1 背景与目标
通过手机聊天软件（飞书），统一管理 PC 上多种交互式/非交互式实例。

**核心目标：**
- **统一管理**：手机端统一控制 PC 上的多种类型实例
- **简洁交互**：指令简洁明了，易于记忆和使用
- **明确反馈**：异常提示包含原因和解决方案
- **纯透传**：消息不做业务处理，直接转发给目标实例
- **无鉴权**：全程无需身份验证，简化使用流程

### 1.2 术语定义

| 术语 | 定义 |
|:---|:---|
| 实例（Instance） | 被管理的运行实体，如 @qoder-ai/qodercli 等进程 |
| 实例类型（Type） | 实例的分类标识，每种类型有特定的启动参数和运行模式 |
| 交互式实例 | 启动后持续运行，支持多次接收输入、返回输出的实例 |
| 非交互式实例 | 启动后自动运行、输出结果、自动结束的实例 |
| 运行状态 | 实例的当前状态：运行中（空闲/忙碌）、已退出 |
| 默认实例 | 通过 `/use ID` 设置的当前操作目标实例 |
| 可中断特性 | 实例支持在执行过程中被 `/interrupt` 指令打断 |

### 1.3 部署环境

| 项目 | 说明 |
|:---|:---|
| 网络环境 | 内网部署，无公网 IP |
| 连接方式 | 服务主动连接飞书平台 |
| 消息接入 | `@larksuiteoapi/node-sdk`，WebSocket 长连接 |

## 2. 实例模型

### 2.1 实例属性

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| ID | number | 自增数字，唯一标识，不可修改 |
| 类型 | string | 实例类型标识，如 @qoder-ai/qodercli |
| 类型说明 | string | 类型用途描述（展示在 `/help` 中） |
| 用户备注 | string | 自定义标签，支持修改 |
| 启动参数 | string[] | 类型特定的启动参数 |
| 运行模式 | RunMode | INTERACTIVE(交互式) / NON_INTERACTIVE(非交互式) |
| 运行状态 | Status | IDLE(空闲) / BUSY(忙碌) / EXITED(已退出) |
| 实例特性 | Feature | INTERRUPTIBLE(可中断) / NON_INTERRUPTIBLE(不可中断) |
| 启动时间 | Date | 实例创建时间 |

### 2.2 实例类型定义

#### @qoder-ai/qodercli

| 属性 | 值 |
|:---|:---|
| **运行模式** | 交互式 |
| **是否可中断** | 否 |
| **典型用途** | 大语言模型 CLI 对话 |

**启动参数：**

| 参数 | 说明 | 示例 |
|:---|:---|:---|
| `-w` | 指定工作区目录 | `-w=/path/to/project` |
| `-c` | 继续上次会话 | `-c` |
| `-r` | 恢复指定会话 | `-r=<session-id>` |
| `--allowed-tools` | 允许使用的工具 | `--allowed-tools=READ,WRITE` |
| `--disallowed-tools` | 禁止使用的工具 | `--disallowed-tools=BASH` |
| `--max-turns` | 最大对话轮数 | `--max-turns=10` |
| `--yolo` | 跳过权限检查 | `--yolo` |

**使用示例：**
```
/new @qoder-ai/qodercli 代码助手 -w=/path/to/project
/new @qoder-ai/qodercli 项目分析 -w=/path/to/project --max-turns=20
```

## 3. 运行模式规则

### 3.1 交互式实例

**典型代表**：@qoder-ai/qodercli

**运行规则：**
- 启动后持续运行，支持多次接收输入、返回输出
- 需手动 `/kill ID` 关闭，或异常退出
- 运行中始终显示在 `/list` 中
- **响应分块**：累积响应内容，达到阈值或检测到结束信号时发送，消息标明顺序号（如 `[1]`），最后一条标 `[完成]`
- **中断能力**：部分实例类型支持 `/interrupt` 指令

**状态流转：**
```
不存在 → /new → 运行中-忙碌 → 初始化完成 → 运行中-空闲 → send → 运行中-忙碌 → 响应完成 → 运行中-空闲
                                                    ↓
                                         /kill 或异常 → 已退出 → /log
```

### 3.2 非交互式实例

**运行规则：**
- 启动 → 自动运行 → 输出全部结果 → 自动结束
- 运行过程中不接受任何外部输入
- 结束后自动从 `/list` 移除，记入 `/log`

**状态流转：**
```
不存在 → /new → 运行中-忙碌（自动运行）→ 自动结束 → 已退出 → /log
```

## 4. 指令系统

### 4.1 指令速查表

| 指令 | 格式 | 功能描述 |
|:---|:---|:---|
| `/help` | `/help [指令名] [类型名]` | 显示帮助信息 |
| `/new` | `/new <类型> <备注> [参数...]` | 创建新实例 |
| `/list` | `/list` | 列出运行中实例 |
| `/log` | `/log` | 查看历史记录 |
| `/rename` | `/rename <ID> <新备注>` | 修改实例备注 |
| `/use` | `/use <ID>` | 设置默认实例 |
| `/<ID>` | `/<ID> <内容>` | 向指定实例发送消息 |
| 纯文本 | `<任意文本内容>` | 发送给默认实例 |
| `/kill` | `/kill <ID>` | 关闭指定实例 |
| `/killall` | `/killall [类型]` | 批量关闭实例 |
| `/interrupt` | `/interrupt [ID]` | 中断实例执行 |

### 4.2 指令详细规格

##### `/help [指令名] [类型名]`

显示帮助信息，支持查询指定指令或类型的详细说明。

| 格式 | 说明 |
|:---|:---|
| `/help` | 显示所有指令列表 |
| `/help <指令名>` | 显示指定指令的详细说明 |
| `/help new <类型名>` | 显示指定类型的使用方式 |

**示例**：`/help new @qoder-ai/qodercli`

---

##### `/new <类型> <备注> [参数...]`

创建指定类型实例。

| 参数 | 必填 | 说明 |
|:---|:---|:---|
| `类型` | 是 | 实例类型标识 |
| `备注` | 是 | 用户自定义标签 |
| `参数` | 否 | 类型特定的启动参数 |

**响应流程：**
1. 系统创建实例进程，分配 ID
2. 实例进入 **BUSY** 状态（启动中）
3. 捕获初始输出，随创建成功消息返回
4. 初始化完成后转为 **IDLE** 状态

**示例**：`/new @qoder-ai/qodercli 代码助手 -w=/path/to/project`

---

##### `/list`

列出所有运行中的实例。

**输出格式**：
```
[ID:1] @qoder-ai/qodercli | 代码助手 | 空闲 | 不可中断 | 运行5分钟
```

---

##### `/log`

查看最近已退出实例记录。

**输出格式**：
```
[ID:1] @qoder-ai/qodercli | 代码助手 | 2026-03-05 14:30:25 | 正常结束
```

---

##### `/rename <ID> <新备注>`

修改指定实例的备注。

**示例**：`/rename 1 AI助手`

---

##### `/use <ID>`

设置默认操作实例。

**示例**：`/use 1`

---

##### `/<ID> <内容>`

向指定 ID 实例发送交互内容。

**约束**：实例必须存在、运行中且为交互式

**示例**：`/1 你好，请帮我写一个快速排序算法`

---

##### 纯文本消息

发送给已设置的默认实例。

**示例**：`请帮我优化这段代码`

---

##### `/kill <ID>`

关闭指定实例。

**示例**：`/kill 1`

---

##### `/killall [类型]`

批量关闭实例。

| 格式 | 说明 |
|:---|:---|
| `/killall` | 关闭所有实例 |
| `/killall <类型>` | 关闭指定类型的所有实例 |

---

##### `/interrupt [ID]`

向交互式实例发送中断信号。

**适用场景**：大语言模型正在生成长回复时打断

**参数说明**：`ID` 可选，省略时使用默认实例

**示例**：`/interrupt 1`

## 5. 异常处理

### 5.1 异常提示规范

- **错误原因 + 解决方案**：所有异常提示需包含错误原因和解决方案/示例
- **统一模板**：使用一致的异常提示格式

### 5.2 异常场景汇总

| 异常类型 | 触发条件 | 提示信息 |
|:---|:---|:---|
| 指令不存在 | 指令名不存在 | `未知指令：xxx。使用 /help 查看所有指令` |
| 类型不支持 | 类型不在支持列表 | `不支持的类型！当前支持：@qoder-ai/qodercli` |
| ID不存在 | 实例不存在或已退出 | `实例ID不存在！可通过 /list 查看运行中的实例` |
| 格式错误 | 参数缺失或格式错误 | `格式错误！正确示例：{示例命令}` |
| 非交互式 | 向非交互式实例发送消息 | `该实例为非交互式，不支持接收手动指令` |
| 未设置默认 | 未指定ID且无默认实例 | `未指定实例！可通过 /use ID 设置默认，或 /ID 内容 直接指定` |
| 默认失效 | 默认实例已退出 | `默认实例已失效！请重新通过 /use ID 切换` |
| 不支持中断 | 实例类型不支持中断 | `该实例类型不支持中断操作` |
| 无实例 | 没有运行中的实例 | `暂无运行中的实例` |
| 无记录 | 没有已退出实例记录 | `暂无已退出实例记录` |

### 5.3 全局规则

| 规则 | 说明 |
|:---|:---|
| 非交互式限制 | 非交互式实例拒绝接收任何手动输入 |
| 自动清理 | 实例退出后自动从 `/list` 移除，记录至 `/log` |
| 状态同步 | 实例状态变化实时同步到 `/list` |
| 无鉴权 | 全程无鉴权、无用户区分，纯透传 |
| 指令容错 | 支持多空格合并、大小写自动转小写 |

## 6. 非功能性需求

### 6.1 性能要求

| 指标 | 默认值 | 说明 |
|:---|:---|:---|
| 最大实例数 | 50 | 并发实例数限制 |
| 日志保留数 | 100 | 已退出实例记录保留数 |

### 6.2 可用性要求

| 要求 | 说明 |
|:---|:---|
| 容错性 | 优雅处理用户输入错误，提供清晰的错误提示 |
| 一致性 | 相同操作在不同场景下产生一致的结果 |
| 可恢复性 | 实例异常退出时正确记录日志并清理资源 |

### 6.3 扩展性要求

**类型扩展**
- 当前仅实现 `@qoder-ai/qodercli` 类型
- 后续可扩展其他类型（如 ssh、node、python 等）
- 新增类型需定义：运行模式、中断特性、启动参数解析、响应处理逻辑

**命令扩展**
- 支持注册新的指令处理器

**平台扩展**
- 消息接入层可适配不同聊天平台（微信、钉钉等）
