# 移动端聊天控制多类型实例系统 —— 需求规格说明书

## 1. 项目概述

### 1.1 项目背景
通过手机聊天软件（飞书），统一管理 PC 上多种交互式/非交互式实例，实现指令简洁、异常提示明确、全程无鉴权、纯透传交互。

### 1.2 核心目标
- **统一管理**：通过手机端统一控制 PC 上的多种类型实例
- **简洁交互**：指令简洁明了，易于记忆和使用
- **明确反馈**：异常提示包含原因和解决方案
- **纯透传**：消息不做业务处理，直接转发给目标实例
- **无鉴权**：全程无需身份验证，简化使用流程

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| 实例（Instance） | 被管理的运行实体，如 qodercli、node、python、ssh 等进程 |
| 实例类型（Type） | 实例的分类标识，每种类型有特定的启动参数和运行模式 |
| 交互式实例 | 启动后持续运行，支持多次接收输入、返回输出的实例 |
| 非交互式实例 | 启动后自动运行、输出结果、自动结束的实例 |
| 运行状态 | 实例的当前状态：运行中（空闲/忙碌）、已退出 |
| 默认实例 | 通过 `/use ID` 设置的当前操作目标实例 |
| 可中断特性 | 实例支持在执行过程中被 `/interrupt` 指令打断 |

## 2. 实例模型

### 2.1 实例属性

每个实例包含以下核心属性：

| 字段 | 类型 | 说明 |
|------|------|------|
| ID | number | 自增数字，唯一、不可修改，作为实例核心标识 |
| 类型 | string | 如 qodercli、node、python、ssh 等 |
| 类型说明 | string | 简短描述该类型用途（展示在 `/help` 中） |
| 用户备注 | string | 自定义用途标签，支持后期修改 |
| 启动参数 | string[] | 各类型实例自定义，创建时传入 |
| 运行模式 | RunMode | 交互式(INTERACTIVE) / 非交互式(NON_INTERACTIVE) |
| 运行状态 | InstanceStatus | 运行中-空闲(IDLE) / 运行中-忙碌(BUSY) / 已退出(EXITED) |
| 实例特性 | Feature | 可中断(INTERRUPTIBLE) / 不可中断(NON_INTERRUPTIBLE) |
| 启动时间 | Date | 实例创建时间 |

### 2.2 实例类型定义

| 实例类型 | 运行模式 | 是否可中断 | 典型用途 | 启动参数示例 |
|----------|----------|-----------|----------|--------------|
| qodercli | 交互式 | 是 | 大语言模型 CLI 对话 | `qodercli chat` |
| ssh | 交互式 | 否 | 远程服务器终端 | `ssh user@host` |
| node | 非交互式 | 否 | 运行 Node.js 脚本 | `script.js [args...]` |
| python | 非交互式 | 否 | 运行 Python 脚本 | `script.py [args...]` |

## 3. 运行模式规则

### 3.1 交互式实例

**典型代表**：qodercli、ssh

**运行规则**：
1. 启动后持续运行，支持多次接收输入、返回输出
2. 需手动 `/kill ID` 关闭，或异常退出
3. 运行中始终显示在 `/list` 中
4. **默认采用"一来一回"模式**：发送消息后进入忙碌状态，等待响应完成后才能发送下一条
5. **中断能力（可选）**：部分实例类型支持 `/interrupt` 指令，可中断当前处理并返回最新状态

**状态流转**：
```
不存在 → /new → 运行中-空闲 → send → 运行中-忙碌 → 响应完成 → 运行中-空闲
                                           ↓
                              /kill 或异常 → 已退出 → /log
```

### 3.2 非交互式实例

**典型代表**：node、python

**运行规则**：
1. 启动 → 自动运行 → 输出全部结果 → 自动结束
2. 运行过程中不接受任何外部输入（`/ID 内容`/纯文本均拒绝）
3. 结束后自动从 `/list` 移除，记入 `/log`（退出原因标记为"正常结束"）

**状态流转**：
```
不存在 → /new → 运行中-忙碌（自动运行）→ 自动结束 → 已退出 → /log
                      ↑
                   拒绝send
```

## 4. 指令系统

### 4.1 指令概览

| 指令 | 功能描述 | 适用场景 |
|------|----------|----------|
| `/help` | 显示帮助信息 | 查看所有指令用法和类型说明 |
| `/new` | 创建新实例 | 启动指定类型的实例 |
| `/list` | 列出运行中实例 | 查看当前所有运行中的实例 |
| `/log` | 查看历史记录 | 查看已退出实例的记录 |
| `/rename` | 修改实例备注 | 给实例设置新的备注名称 |
| `/use` | 设置默认实例 | 设置后续操作的默认目标实例 |
| `/ID` | 向指定实例发送消息 | 与交互式实例进行通信 |
| 纯文本 | 发送给默认实例 | 快捷与默认实例通信 |
| `/kill` | 关闭指定实例 | 终止单个实例 |
| `/killall` | 批量关闭实例 | 终止所有或指定类型的实例 |
| `/interrupt` | 中断实例执行 | 打断正在处理的任务 |

### 4.2 指令详细规格

#### 4.2.1 `/help`

**功能**：返回所有指令用法 + 支持的实例类型（含类型说明 + 特性标签） + 示例命令

**格式**：`/help`

**输出内容**：
- 所有指令的用法说明
- 支持的实例类型列表（含类型说明）
- 类型特性标签（如「可中断」「不可中断」）
- 常用命令示例

**异常场景**：无（该指令无入参，始终返回完整帮助信息）

---

#### 4.2.2 `/new 类型 备注 [参数...]`

**功能**：创建指定类型实例，参数由对应类型定义

**格式**：`/new <类型> <备注> [参数...]`

**参数说明**：
- `类型`：实例类型标识（如 qodercli、node、python、ssh）
- `备注`：用户自定义的实例标签（用于识别用途）
- `参数`：类型特定的启动参数（可选，根据类型需求）

**示例**：
- `/new qodercli 代码助手 chat`
- `/new node 数据处理 script.js data.json`
- `/new python 数据分析 analysis.py --input data.csv`

**成功响应**：返回创建成功的实例 ID 和基本信息

**异常场景**：

| 异常类型 | 触发条件 | 提示信息 |
|----------|----------|----------|
| 格式错误 | 缺失「类型」或「备注」 | `格式错误！正确示例：/new qodercli 代码助手 chat` |
| 类型不支持 | 类型不在支持列表中 | `不支持的类型！当前支持：qodercli, node, python, ssh` |
| 参数错误 | 参数格式不符合类型要求 | `参数错误！该类型参数示例：/new node 任务 script.js` |

---

#### 4.2.3 `/list`

**功能**：列出所有运行中的实例

**格式**：`/list`

**输出内容**：
| 字段 | 说明 |
|------|------|
| ID | 实例唯一标识 |
| 类型 | 实例类型 |
| 备注 | 用户自定义标签 |
| 状态 | 空闲/忙碌 + 耗时 |
| 特性 | 可中断/不可中断 |

**示例输出**：
```
运行中的实例：
[ID:1] qodercli | 代码助手 | 空闲 | 可中断 | 运行5分钟
[ID:2] ssh | 服务器A | 忙碌(2分钟) | 不可中断 | 运行10分钟
```

**异常场景**：

| 异常类型 | 触发条件 | 提示信息 |
|----------|----------|----------|
| 无实例 | 没有运行中的实例 | `暂无运行中的实例` |

---

#### 4.2.4 `/log`

**功能**：查看最近已退出实例记录

**格式**：`/log`

**输出内容**：
| 字段 | 说明 |
|------|------|
| 原ID | 实例原唯一标识 |
| 类型 | 实例类型 |
| 备注 | 用户自定义标签 |
| 退出时间 | 实例结束运行的时间 |
| 退出原因 | 正常结束/异常退出/手动终止 |

**示例输出**：
```
已退出实例记录：
[ID:1] node | 数据处理 | 2026-03-05 14:30:25 | 正常结束
[ID:2] python | 数据分析 | 2026-03-05 14:15:10 | 手动终止
```

**异常场景**：

| 异常类型 | 触发条件 | 提示信息 |
|----------|----------|----------|
| 无记录 | 没有已退出的实例记录 | `暂无已退出实例记录` |

---

#### 4.2.5 `/rename ID 新备注`

**功能**：修改指定实例的备注

**格式**：`/rename <ID> <新备注>`

**参数说明**：
- `ID`：目标实例的数字标识
- `新备注`：新的用户自定义标签

**示例**：`/rename 1 AI助手`

**成功响应**：返回修改成功的确认信息

**异常场景**：

| 异常类型 | 触发条件 | 提示信息 |
|----------|----------|----------|
| ID不存在 | 实例ID不存在或已退出 | `实例ID不存在！可通过 /list 查看运行中的实例` |
| 格式错误 | 缺失「新备注」 | `格式错误！正确示例：/rename 1 新备注` |

---

#### 4.2.6 `/use ID`

**功能**：设置默认操作实例，后续无 ID 前缀的纯文本消息默认发给该实例

**格式**：`/use <ID>`

**参数说明**：
- `ID`：目标实例的数字标识

**示例**：`/use 1`

**成功响应**：返回设置成功的确认信息

**异常场景**：

| 异常类型 | 触发条件 | 提示信息 |
|----------|----------|----------|
| ID不存在 | 实例ID不存在或已退出 | `实例ID不存在！可通过 /list 查看运行中的实例` |
| 格式错误 | 缺失「ID」 | `格式错误！正确示例：/use 1` |

---

#### 4.2.7 `/ID 内容`

**功能**：向指定 ID 实例发送交互内容

**格式**：`/<ID> <内容>`

**参数说明**：
- `ID`：目标实例的数字标识
- `内容`：要发送给实例的消息内容

**允许发送条件**：实例存在且运行中 + 实例为交互式

**示例**：`/1 你好，请帮我写一个快速排序算法`

**成功响应**：返回实例的输出响应

**异常场景**：

| 异常类型 | 触发条件 | 提示信息 |
|----------|----------|----------|
| ID不存在 | 实例ID不存在或已退出 | `实例ID不存在！可通过 /list 查看运行中的实例` |
| 非交互式 | 实例为非交互式 | `该实例为非交互式，不支持接收手动指令` |
| 格式错误 | 缺失「内容」 | `格式错误！正确示例：/1 你好` |

---

#### 4.2.8 纯文本消息（无 `/ID` 前缀）

**功能**：发送给已设置的默认实例

**格式**：`<任意文本内容>`

**示例**：`请帮我优化这段代码`

**成功响应**：返回默认实例的输出响应

**异常场景**：

| 异常类型 | 触发条件 | 提示信息 |
|----------|----------|----------|
| 未设置默认 | 未设置默认实例 | `未指定实例！可通过 /use ID 设置默认，或 /ID 内容 直接指定` |
| 默认失效 | 默认实例已退出 | `默认实例已失效！请重新通过 /use ID 切换` |
| 非交互式 | 默认实例为非交互式 | `默认实例为非交互式，不支持接收手动指令` |

---

#### 4.2.9 `/kill ID`

**功能**：关闭指定实例

**格式**：`/kill <ID>`

**参数说明**：
- `ID`：目标实例的数字标识

**示例**：`/kill 1`

**成功响应**：返回实例已终止的确认信息

**异常场景**：

| 异常类型 | 触发条件 | 提示信息 |
|----------|----------|----------|
| ID不存在 | 实例ID不存在或已退出 | `实例ID不存在！可通过 /list 查看运行中的实例` |
| 格式错误 | 缺失「ID」 | `格式错误！正确示例：/kill 1` |

---

#### 4.2.10 `/killall [类型]`

**功能**：无参数时关闭所有实例，带参数时仅关闭指定类型的所有实例

**格式**：`/killall [类型]`

**参数说明**：
- `类型`（可选）：指定要关闭的实例类型

**示例**：
- `/killall` - 关闭所有实例
- `/killall node` - 关闭所有 node 类型实例

**成功响应**：返回已关闭实例的数量和列表

**异常场景**：

| 异常类型 | 触发条件 | 提示信息 |
|----------|----------|----------|
| 无实例 | 没有运行中的实例 | `暂无运行中的实例可关闭` |
| 类型不支持 | 指定类型不在支持列表中 | `不支持的类型！当前支持：qodercli, node, python, ssh` |

---

#### 4.2.11 `/interrupt [ID]`

**功能**：向指定交互式实例发送中断信号

**适用场景**：如大语言模型 CLI 正在生成长回复，用户希望打断并重新输入

**格式**：`/interrupt [ID]`

**参数说明**：
- `ID`（可选）：目标实例的数字标识，省略时使用默认实例

**ID 省略规则**：未指定 ID 时，向默认实例发送中断信号

**示例**：
- `/interrupt 1` - 中断 ID 为 1 的实例
- `/interrupt` - 中断默认实例

**成功响应**：返回中断成功的确认信息

**异常场景**：

| 异常类型 | 触发条件 | 提示信息 |
|----------|----------|----------|
| ID不存在 | 指定ID不存在或已退出 | `实例ID不存在！可通过 /list 查看运行中的实例` |
| 未设置默认 | 未指定ID且未设置默认实例 | `未指定实例！可通过 /use ID 设置默认，或 /interrupt ID 直接指定` |
| 默认失效 | 默认实例已退出 | `默认实例已失效！请重新通过 /use ID 切换` |
| 非交互式 | 目标实例为非交互式 | `该实例为非交互式，不支持中断操作` |
| 不支持中断 | 实例类型不支持中断 | `该实例类型不支持中断操作` |

## 5. 全局规则

### 5.1 异常提示规范

1. **错误原因 + 解决方案**：所有指令异常提示需包含「错误原因 + 解决方案/示例」，而非仅告知"错误"
2. **统一模板**：使用统一的异常提示模板，确保用户体验一致性

### 5.2 实例生命周期规则

1. **非交互式实例限制**：非交互式实例一律拒绝接收任何手动输入（`/ID 内容` 或纯文本消息）
2. **自动清理**：实例退出后自动从 `/list` 移除，并记录退出时间、原因至 `/log`
3. **状态同步**：实例状态变化实时同步到 `/list` 显示

### 5.3 交互规则

1. **无鉴权**：全程无鉴权、无用户区分，所有操作纯透传
2. **指令容错**：指令支持基础容错：
   - 多空格自动合并
   - 大小写自动转为小写

### 5.4 异常提示模板汇总

| 场景 | 提示模板 |
|------|----------|
| ID不存在 | `实例ID不存在！可通过 /list 查看运行中的实例` |
| 格式错误 | `格式错误！正确示例：{示例命令}` |
| 类型不支持 | `不支持的类型！当前支持：{类型列表}` |
| 非交互式 | `该实例为非交互式，不支持接收手动指令` |
| 未设置默认 | `未指定实例！可通过 /use ID 设置默认，或 /ID 内容 直接指定` |
| 默认失效 | `默认实例已失效！请重新通过 /use ID 切换` |
| 不支持中断 | `该实例类型不支持中断操作` |

## 6. 非功能性需求

### 6.1 性能要求

| 指标 | 约束 | 说明 |
|------|------|------|
| 命令响应时间 | < 100ms | 纯内存操作应在100ms内完成 |
| 实例创建时间 | < 500ms | 进程创建应在500ms内完成 |
| 消息转发延迟 | < 50ms | 消息透传应在50ms内完成 |
| 最大实例数 | 可配置 | 默认限制50个并发实例 |
| 日志保留数 | 可配置 | 默认保留最近100条 |

### 6.2 可用性要求

1. **容错性**：系统应能优雅处理用户输入错误，提供清晰的错误提示
2. **一致性**：相同操作在不同场景下应产生一致的结果
3. **可恢复性**：实例异常退出时，应正确记录日志并清理资源

### 6.3 扩展性要求

1. **类型扩展**：支持通过插件机制新增实例类型，无需修改核心代码
2. **命令扩展**：支持注册新的指令处理器
3. **平台扩展**：消息接入层可适配不同聊天平台（微信、钉钉等）
